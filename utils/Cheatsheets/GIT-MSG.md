# Common Commit Labels(Types)
Label | Purpose | Example
--- | --- | ---
feat | New feature or functionality | feat(score): add multi-player support
fix | Bug fix | fix(ui): resolve button click delay
test | Adding or updating tests | test(api): add unit tests for authentication
refactor | Code improvement (without changing behavior) | refactor(db): optimize query execution
docs | Documentation updates | docs(readme): update installation steps
style | Code formatting, linting (no logic changes) | style(theme): fix spacing inconsistencies
chore | Maintenance, dependencies, or config changes | chore(gradle): update dependencies
perf | Performance improvement | perf(animations): optimize frame rate handling
ci | Changes to CI/CD configuration	| ci(jenkins): add deployment script
build | Changes affecting the build system	build(gradle): fix build warnings
revert | Revert a previous commit | revert: undo score calculation changes
